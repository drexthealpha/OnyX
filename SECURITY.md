# Security Policy

### Reporting security problems in the OnyX Protocol

**DO NOT CREATE A GITHUB ISSUE** to report a security problem.

Instead please use the Report a Vulnerability link or email security@onyx.os.
Provide a helpful title, detailed description of the vulnerability and an exploit
proof-of-concept. Speculative submissions without proof-of-concept will be closed
with no further consideration.

If you haven't done so already, please **enable two-factor auth** in your GitHub account.

Expect a response as fast as possible, typically within 72 hours.

If you do not receive a response within the SLA, send an email to
security-escalations@onyx.os with the full URL of the advisory you have created. DO NOT
include attachments or provide detail sufficient for exploitation regarding the
security issue in this email. **Only provide such details in the original advisory**.

If you do not receive a response from escalations please followup with
the team directly. You can do this in the `#security` channel of the
OnyX discord server, by pinging the `Security Team` role in the channel and referencing the fact that you submitted a security problem.
