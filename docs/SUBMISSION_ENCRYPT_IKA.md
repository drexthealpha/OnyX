# Encrypt / Ika Track Submission — Non-Custodial MPC Signing

**Track:** FHE + dWallet Cross-Chain Signing
**Packages:** `@onyx/bridge` · `@onyx/fhe` · `@onyx/vault`

## 🔒 The Problem
Traditional autonomous agents often store private keys in plaintext `.env` files or in-memory, creating a massive single point of failure and a primary target for attackers.

## 🔑 The ONYX Solution
ONYX integrates the **dWallet network via Ika** to provide agents with fully non-custodial signing capabilities. The agent's key is sharded across the network using MPC, ensuring that no single entity (including the agent owner) has the full key.

- **FHE Vaults:** Encrypted storage for agent state and secrets.
- **dWallet Signing:** Secure, cross-chain transaction approval without local keys.
- **Privacy First:** Total operational security for autonomous financial activities.
