# Crosspost

Adapt one piece of content across multiple platforms without losing the author's authentic voice.

## When to Activate

- Taking one article and distributing to multiple social platforms
- Repurposing a tweet into a LinkedIn post
- Converting a blog post into short-form content
- Building a launch sequence across platforms

## Core Workflow

### 1. Identify the Source Value
Extract the single strongest claim, insight, or artifact from the source material. Don't try to capture everything.

### 2. Adapt per Platform Rules
Each platform has different constraints. Write for each platform's native format, not a generic template.

### 3. Preserve Voice
The adaptation should sound like the same author on each platform — not a corporate marketing voice.

## Platform Rules Summary

| Platform | Max Length | Best For | Avoid |
|----------|-----------|----------|-------|
| X | 280 chars / 25K threads | Real-time, threads, hot takes | Corporate speak |
| LinkedIn | 3,000 chars | Professional insights, long-form | Inspiration porn |
| Threads | 500 chars | Casual, visual content | Long threads |
| Bluesky | 300 chars | Open discussion | Engagement bait |

## Platform Adaptation Examples

From source: "ONYX built a full AI OS on Solana. Zero operator cost. Every API key belongs to the user."

- **X**: "We built a full AI OS on Solana. Zero operator cost. Every API key belongs to the user. Not a feature. The architecture."
- **LinkedIn**: "Sovereign AI OS on Solana. Architecture decision: we don't subsidize API costs. Every key belongs to the user. Result: sustainable, user-owned AI infrastructure."
- **Threads**: (visual card with key stats)
- **Bluesky**: Same as X, but without the thread structure

## Hashtag Strategy

- X: 1-2 relevant hashtags at end
- LinkedIn: 0-3, integrated naturally
- Threads: 1-2 on opening or closing
- Bluesky: Optional, topic-tagged

## Character Counting Tips

For platforms with limits:
- Count actual visible characters, not code points
- Account for URL previews (they consume ~23 chars)
- For threads, reserve ~20 chars for numbering (1/X)

## Posting Sequence

Typical launch order:
1. X thread (first, fastest)
2. LinkedIn (within 1 hour)
3. Threads/Bluesky (within 2 hours)
4. Newsletter (same day or next day)
